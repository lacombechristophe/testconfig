Le fichier .env.example est un modele. Il faut le renommer en .env (ou .env.local) pour qu'il soit pris en compte en local. Ne jamais publier le fichier .env sur GitHub (il contient des cles secretes).
Les variables CONTACT_* controlent les informations affichees dans les emails de confirmation envoyes aux prospects. Si elles ne sont pas renseignees, les valeurs par defaut sont utilisees.

C. Rate limiting (recommande)

Pour un rate limiting persistant, activer Vercel KV :

  1. Vercel Dashboard → Storage → Create KV Store
  2. Lier le store au projet configurateur
  3. Les variables KV_REST_API_URL et KV_REST_API_TOKEN sont injectees automatiquement

2. Modifier les informations du site

Toutes les informations modifiables du configurateur sont centralisees dans un seul fichier :

config.js

Comment modifier sur GitHub

  1. Ouvrir le depot GitHub du projet
  2. Cliquer sur le fichier config.js
  3. Cliquer sur l'icone crayon (modifier)
  4. Changer les valeurs souhaitees (sans toucher aux guillemets ni aux points-virgules)
  5. Cliquer sur Commit changes
Le site se met a jour automatiquement en quelques secondes apres chaque modification.

Si vous changez le nom ou le telephone, pensez a mettre a jour les deux.

3. Exemple : changer un prix

Pour modifier le prix de base du Coverseal Automatique de 13 890 € a 14 500 € :

  1. Ouvrir config.js sur GitHub
  2. Trouver la ligne auto: 13890,
  3. Remplacer par auto: 14500,
  4. Cliquer sur Commit changes

Le nouveau prix sera visible sur le configurateur en quelques secondes.

4. Exemple : ajouter une couleur membrane

Pour ajouter une nouvelle couleur (ex: Gris anthracite RAL 7016 avec supplement 459 €) :

  1. Ouvrir config.js sur GitHub
  2. Dans la section DISKOOV_MEMBRANE_COLORS, ajouter une ligne avant le ] :

{ h: '#4A5568', r: 'RAL 7016', l: 'Gris anthracite', delta: 459 },

h = code couleur hexadecimal, r = code RAL, l = nom en francais, delta = supplement en euros.

5. Integration WordPress

Le configurateur s'integre dans WordPress via un iframe. Coller ce code dans l'editeur de la page WordPress :

<style> .diskoov-iframe-wrap { position: relative; width: 100%; height: 100svh; overflow: hidden; } .diskoov-iframe-wrap iframe { position: absolute; inset: 0; width: 100%; height: 100%; border: none; } </style> <div class="diskoov-iframe-wrap"> <iframe src="https://configurateur.diskoov.fr" title="Configurateur piscine Diskoov" allow="autoplay; encrypted-media" loading="lazy"> </iframe> </div>
Pour masquer le header/footer WordPress sur cette page, ajouter dans functions.php un filtre qui cache ces elements sur la page configurateur.

7. Securite en place

Le configurateur integre les protections suivantes :

  • HTTPS force sur toutes les pages (HSTS)
  • Validation complete des donnees cote serveur (email, telephone, dimensions)
  • Protection anti-spam : rate limiting (1 demande/15 sec par email, 10/heure par IP)
  • Protection anti-bot : champ honeypot invisible
  • CORS strict : seuls diskoov.fr et configurateur.diskoov.fr sont autorises
  • Echappement HTML dans tous les emails (empeche les injections)
  • Polices auto-hebergees : aucun appel a Google Fonts (conforme RGPD)
  • Banniere cookies CNIL : activee automatiquement si Google Analytics est configure
  • Retry automatique : si un lead echoue (reseau), il est sauvegarde et renvoye automatiquement
